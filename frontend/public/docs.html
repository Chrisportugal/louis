Louis - Yield Agent on HyperEVM

Louis is an AI-powered yield vault on HyperEVM. Deposit stablecoins and the vault automatically allocates your capital across the highest-yielding lending protocols.

Why Louis? Instead of manually comparing APYs across protocols, Louis monitors rates in real-time and rebalances to capture the best yields on HyperEVM.

How It Works

  1. Deposit — Deposit USDC, USDT, or other supported stablecoins. The vault handles any necessary conversions. You receive louisUSD share tokens representing your position.
  2. AI Allocates — The AI agent monitors APYs across all integrated lending protocols and allocates your deposit to the best one.
  3. Auto-Rebalance — Every 30 minutes, the agent checks for better opportunities. If it finds one with >0.5% higher APY, it moves funds automatically.
  4. Withdraw Anytime — Redeem your louisUSD shares for the underlying stablecoins at any time. No lock-up period.

Risks

  • Smart contract risk — The vault is not upgradeable. Bugs cannot be patched in-place, but also cannot be exploited via upgrades. The owner can emergencyPull() and pause() to protect deposits.
  • Protocol risk — Louis deposits into third-party protocols. If an underlying protocol is exploited, funds there could be lost.
  • Centralization risk — The vault owner can add protocols and emergency withdraw. The allocator can move funds. If either private key is compromised, funds could be at risk.
  • APY risk — The AI agent relies on on-chain data and APIs. If sources report incorrect rates, it could reallocate to a suboptimal protocol. APY is variable.
  • Swap risk — USDC/USDT deposits are swapped via HyperSwap. In extreme conditions, USDHL could depeg, causing higher slippage. If DEX liquidity dries up, deposits may fail.
  • Chain risk — HyperEVM (Chain ID 999) is a newer chain with less battle-testing than Ethereum mainnet.
Build

Architecture

Stack: Solidity 0.8.24 (Foundry) • TypeScript (Viem) • React (Vite + wagmi) • OpenZeppelin v5

Smart Contract

YieldVault.sol is an ERC-4626 tokenized vault with multi-protocol support. Two protocol interfaces:

  • AAVE_V3 — IPool.supply() / withdraw(), aToken balance tracking
  • ERC4626_VAULT — IERC4626.deposit() / withdraw(), share-to-asset conversion

AI Agent

Off-chain TypeScript agent (runs every 30 min). Fetches APYs from on-chain reads + Morpho API, calls reallocate() when a better opportunity exists (>0.5% threshold), and harvest() to collect fees.

Role Separation

Security

  • Reentrancy guard — All state-changing functions use OpenZeppelin ReentrancyGuard
  • Pausable — Owner can pause deposits. Withdrawals always remain open.
  • Minimum deposit — 0.001 USDHL minimum to prevent inflation attacks
  • Zero-address checks — Constructor and all admin setters validate inputs
  • Emergency pull — emergencyPull(index) instantly withdraws from any protocol
  • Non-upgradeable — Immutable once deployed. No proxy, no admin backdoor.
  • 54 test cases — Full coverage of deposits, withdrawals, reallocations, fees, security, and edge cases

Developers

SDK — Integrate Louis

Use the Louis SDK to add vault deposits/withdrawals to any frontend or protocol. Built on viem.

npm install viem

Read Vault Data (no wallet needed)

import { LouisSDK } from './sdk'

const louis = new LouisSDK()

// Get total vault TVL
const tvl = await louis.totalAssets()
console.log(`TVL: $${tvl}`)

// Get full vault state with protocol allocations
const state = await louis.getVaultState()
console.log(state.protocols)

// Get a user's position
const pos = await louis.getUserPosition('0xUSER...')
console.log(`Shares: ${pos.shares}, Value: $${pos.assets}`)

Deposit (with wallet)

import { LouisSDK, ADDRESSES } from './sdk'
import { createWalletClient, custom } from 'viem'

// Connect user's wallet (e.g. from Rabby, MetaMask)
const walletClient = createWalletClient({
  transport: custom(window.ethereum),
})

const louis = new LouisSDK({ walletClient })

// Direct USDHL deposit
await louis.approve(ADDRESSES.USDHL, ADDRESSES.VAULT, '100')
await louis.deposit('100', userAddress)

// Or deposit USDC/USDT (auto-swaps to USDHL)
const txs = await louis.zapDeposit(ADDRESSES.USDC, '100', userAddress)
// Returns tx hashes for each step: approve → swap → approve → deposit

Withdraw

// Withdraw by asset amount
await louis.withdraw('50', userAddress, userAddress)

// Or redeem all shares
const pos = await louis.getUserPosition(userAddress)
await louis.redeem(pos.shares, userAddress, userAddress)

Get Swap Quote

const quote = await louis.getSwapQuote(ADDRESSES.USDC, '1000')
console.log(`Expected: ${quote.expectedOutput} USDHL`)
console.log(`Min (0.5% slippage): ${quote.minimumOutput} USDHL`)
console.log(`Price impact: ${quote.priceImpact}%`)
Integration example: To add Louis to a protocol like Harmonix, import the SDK, call louis.getVaultState() to show TVL/APY, and use louis.zapDeposit() to let users deposit from any stablecoin.

AI Agent

Status: Live ✅ The agent runs 24/7 on a dedicated server, checking yields every 30 minutes.

The yield agent is a standalone TypeScript process that monitors APYs across all integrated protocols, rebalances to the highest yield, and harvests fees automatically.

How It Works

  1. Every 30 minutes, fetches live APYs from on-chain data + Morpho API
  2. Compares current protocol yield against the best available
  3. If a better protocol exceeds the current by >0.5%, calls reallocate()
  4. If accumulated yield exceeds $1, calls harvest() to collect fees

Security Model

The agent uses a dedicated wallet separate from the vault owner. This wallet can only:

  • reallocate() — Move funds between already-approved protocols
  • setActiveIndex() — Change which protocol receives new deposits
  • harvest() — Collect management fees

It cannot add/remove protocols, pause the vault, emergency withdraw, or access user funds. Even if the server is compromised, the worst case is funds moving between legitimate protocols.

Running Your Own

# Dry run (logs what it would do, no transactions)
PRIVATE_KEY=0x... DRY_RUN=1 npx tsx src/agent.ts

# Live mode
PRIVATE_KEY=0x... npx tsx src/agent.ts

# Run 24/7 with pm2
npm install -g pm2
PRIVATE_KEY=0x... pm2 start "npx tsx src/agent.ts" --name louis-agent
pm2 save && pm2 startup
