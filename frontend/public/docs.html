Louis - Yield Agent on HyperEVM

Louis is an AI-powered yield allocation vault on HyperEVM that automatically deploys capital to the highest-yielding lending protocols. Deposit your stablecoins, and the AI agent handles the rest.

Why Louis? Instead of manually comparing APYs across protocols, Louis monitors rates in real-time and rebalances your deposits to capture the best risk-adjusted yields on HyperEVM.

How It Works

  1. Deposit - You deposit USDHL (or other supported stablecoins) into the Louis vault. You receive louisUSD share tokens representing your position.
  2. AI Monitors - The AI agent continuously monitors APYs across all integrated lending protocols on HyperEVM.
  3. Auto-Reallocate - When a better opportunity is found, the agent calls reallocate() to move funds to the higher-yielding protocol.
  4. Withdraw Anytime - Redeem your louisUSD shares at any time. The vault automatically pulls funds from protocols to fulfil your withdrawal.

Multi-Stablecoin Deposits

Louis accepts USDHL, USDC, and USDT. When you deposit a non-USDHL token, the frontend automatically swaps it to USDHL via HyperSwap (Uniswap V2 fork) before depositing into the vault.

Swap Flow: Approve token → Swap to USDHL via HyperSwap → Approve USDHL → Deposit into vault. Default slippage: 0.5%. Quotes refresh every 15 seconds.

Management Fee

Louis charges a 10% management fee on earned yield (not on deposits). The fee is collected via the harvest() function, which:

  1. Calculates yield earned since the last harvest (current totalAssets vs snapshot)
  2. Takes 10% of the yield as a fee
  3. Mints new vault shares to the fee recipient, proportional to the fee amount
  4. Updates the snapshot for the next harvest cycle

This is the standard DeFi fee pattern (used by Yearn, Morpho, etc.) — share dilution ensures the fee is taken proportionally without moving underlying assets.

YieldVault Contract

The core contract is an ERC-4626 tokenized vault built on OpenZeppelin v5. It supports two protocol types:

Protocol Types

  • AAVE_V3 - Interacts via IPool.supply() and IPool.withdraw(). Tracks deposits via aToken balances (1:1 with underlying).
  • ERC4626_VAULT - Interacts via IERC4626.deposit() and IERC4626.withdraw(). Tracks deposits via convertToAssets(shares) for accurate accounting with variable share prices.

Role Separation

Smart Contracts

  • YieldVault.sol - ERC-4626 vault with multi-protocol support + 10% management fee
  • Built on OpenZeppelin v5 (ERC4626, Ownable, SafeERC20)
  • Compiled with Solidity 0.8.24

Off-Chain Agent

  • Monitor - Fetches live APYs from on-chain data + Morpho GraphQL API + DefiLlama
  • Ranker - Compares risk-adjusted yields across all protocols
  • Executor - Calls reallocate() via Viem transactions when a better opportunity is found

Security

  • Non-custodial - Users can always withdraw. The vault never locks funds.
  • Emergency pull - Owner can call emergencyPull(index) to instantly pull all funds from any protocol back to idle.
  • Role separation - Owner manages protocols and emergencies. Allocator (AI agent) can only reallocate between existing protocols, not add/remove them.
  • Battle-tested dependencies - Built on OpenZeppelin v5.6.1 and ERC-4626 standard.
  • Comprehensive tests - 41 test cases covering deposits, withdrawals, reallocations, interest accrual, fee harvesting, cross-protocol operations, and edge cases.

Deployment

# Install dependencies
npm install
forge install

# Run tests
forge test -v

# Deploy to HyperEVM
export PRIVATE_KEY=your_private_key_here
forge script contracts/script/Deploy.s.sol \
  --rpc-url https://rpc.hyperliquid.xyz/evm \
  --broadcast

# Run the yield monitor
npx tsx src/monitor.ts

# Start the frontend
cd frontend && npm run dev

Contract API Reference

deposit(assets, receiver)

Deposits assets amount of underlying tokens and mints vault shares to receiver. The deposited tokens are immediately supplied to the active protocol.

withdraw(assets, receiver, owner)

Burns shares from owner and sends assets underlying tokens to receiver. Automatically pulls from protocols if the vault has insufficient idle balance.

reallocate(from, to, amount)

Moves amount of underlying tokens from protocol at index from to protocol at index to. Only callable by the allocator (AI agent) or owner.
